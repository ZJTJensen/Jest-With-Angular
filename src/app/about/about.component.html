PROS
Easy Ui Testing

Snapshot Testing Allows for a easy of use way to test your UI files.
Built with Jasmine in mind

Jest was created to act ontop of Jasmine and as such, most of the code is similar, if not identicle, to Jasmine.
Runs Fast

"2.5x faster than with Karma".
CONS
Built for React

Jest, the platform snapshot testing was built on, was created for React and not Angular.
Little Documentation

As such There is very litle when it comes to documentation for implementing jest in angular. And the main sites documentation is soley for react.
In Depth Analysis

You analyse your failing html snapshots in depth, and not just accept the changes made.
